Administration



Für folgende Administrationsaufgaben gibt es Dialoge:

Das Release-Management verwaltet die Releases eines Szenarios. Einerseits wird eine Übersicht über die Artefakte eines Release gegeben. Außerdem kann der Status eines Release geändert werden und die Erzeugung der Release-Artefakte angestoßen werden.
Über das Node-Management können Serverknoten registriert und verwaltet werden.
Auf der Deployment-Seite wird dargestellt, welche Releases auf welchen Server-Knoten deployt sind. Deployments auf Entwicklungsumgebungen und Transporte in andere Umgebungen können angestoßen werden.
Die Benutzerverwaltung wird verwendet, um die Zuordnung der Anwender zu den Scorecards bzw. zum REST Service zu verwalten. Die Zuordnung erfolgt dabei nicht direkt, sondern über Gruppen und Rollen.
Im Rahmen der Mandantenadministration werden die Anwender dieser Anwendung mindestens einem Mandanten zugeordnet. Eine Mandantenzuordnung ist für die Arbeit mit diesem Werkzeug ein zwingende Voraussetzung. Mandantenzuordnungen können nur von dem Owner des Systems durchgeführt werden.
