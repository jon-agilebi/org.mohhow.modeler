Administration



There are dialogues for the following administration tasks:

The Release-Management manages the releases of a given scenario. An overview on all artefacts of a release is given. Furthermore, the status of a release can be changed and the generation of the release artefacts can be started.
Within the Node Management you can register and manage server nodes.
On the Deployment Page you can see, which releases are deployed on which server nodes. You can initiate deployments on development nodes and you can transport releases into other environments.
The User Management is about the link between users and Scorecards and REST Services, respectively. The link is not direct: User can belong to groups, groups are linked to roles and roles are linked to specifications.
In the context of the Client Administration all users are linked to at least one client. If a user is not linked to a client, he cannot work with this web tool.
